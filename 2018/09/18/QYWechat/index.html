部署IIS遇到的问题总结

xuzg 发表于 2018-09-18

前言

第一次部署项目到IIS，出现挺多问题，做个笔记，长个记性~

问题列表

1. 发布到 IIS 访问网站提示 HTTP Error 500.19 - Internal Server Error 配置错误 由于权限不足而无法读取配置文件

问题原因：

- 由于权限不足而无法读取配置文件

解决方法：

- 设置文件夹的操纵权限
    原文件夹，右击选择它的属性，打开“安全”选项卡 ->
    编辑 -> *的权限选项卡 -> 添加 -> 选择组或用户选项卡 -> 高级 -> 立即查找 -> 
    找到Everyone -> 选择并确定 -> 选择Everyone -> 允许这栏设置完全控制 -> 确定
    ----------------------------------------------------------------------
- 同理把配置文件也设置一下操作权限
2. 发布到 iis 访问网站无响应（假死状态）

问题可能原因：

- 独立进程的 内存堆栈消耗完了，IIS不能创建更多的进程工作空间来处理

解决方法：

- 重启 IIS
    最外层用户 -> 重新启动
3. windows server 2008 R2 装 iis

步骤

- 控制面板 -> 管理工具 -> 服务器管理
4. 发布时会有遗漏外部引入的 dll 文件的情况
- 发布后记得检查一下发布出的bin文件夹里有没有, 如果没有则自己手动把dll文件放到该目录之下
5. 调 webservice 的接口最好是用服务引用的方式（我之前是用 HttpWebRequest，本地调用可以，结果生产环境调用失败了）

使用步骤

1. 在解决方案中选中项目，右键添加服务引用
2. 在地址一栏里输入 webservice 的地址，点击前往
3. 找到服务后，点下面的高级按钮，点添加 web 引用
4. 在 URL 一栏中输入 webservice 的地址，点击右边箭头
5. 找到服务后，记住找到 *服务的名字* 和 *web 引入名*，然后点添加引用
6. 现在就可以在你的项目中调用 webservice 了

6. “/”应用程序中的服务器错误

问题原因

- 应用程序池版本与应用的 Framework 版本不兼容

解决方法

- 使用与 应用 Framework 兼容的 应用程序池
- 我这里是使用 Framework 4.0，可以去我的网盘下载，下载地址 https://pan.baidu.com/s/14J8NBrvuHxYQjESEYKvrgw
- 安装成功后，找到你的网站 -> 高级设置 -> 应用程序池 -> 选择 .net v4.0 classic
7. 部署 IIS，打开网站提示 未能加载文件或程序集“Interop.Domino”或它的某一个依赖项。试图加载格式不正确的程序

问题原因

- 部署的机子是 x64的，而 Interop.Domino.dll 是 x32 的版本

解决方法

- 通过更改IIS的应用程序池来解决：
1. 打开iis选择应用程序池
2. 选择你网站使用的应用程序池，点击右边高级设置
3. 启用 32 位应用程序一栏选择 true，确定就OK了~
8. 部署 IIS，打开网站提示 TTP 错误 403.14 - Forbidden，Web 服务器被配置为不列出此目录的内容

问题原因

- 没有为请求的 URL 配置默认文档，并且没有在服务器上启用目录浏览

解决方法

1. 设置默认文档（推荐）
    - 找到你的网站，双击默认文档
    - 点击右边添加，名称填你默认想打开的文件
2. 启用目录浏览
    - 双击目录浏览，点击右边启用
9. IIS 上的网站运行了一段时间后，被自动回收

问题原因

- iis 回收的固定时间间隔为 1740 分钟，意味着网站运行了 1740 分钟后，应用程序池会被回收
- iis 应用程序池闲置超时时间为 20 分钟，意味着20分钟内工作线程没有处理任何请求，也没接受到任何新的请求，则会被自动回收掉

解决方法

1. 找到你网站使用的应用程序池，打开高级设置
2. 将 固定时间间隔 设置为 0
3. 将 闲置超时时间 设置为 0
