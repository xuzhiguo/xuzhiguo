问题背景

由于之前小程序很多数据都是写死的，今天想把数据通过接口获得。所以我在小程序中建立了如下请求：

结果发现报错了，提示：

VM1560:1 thirdScriptError
Unexpected token o in JSON at position 1;at pages/takeOut/takeOut getOrderMenu function;at api request success callback function
SyntaxError: Unexpected token o in JSON at position 1

看了一下错误信息，大概意思是JSON解析错误吧。然后看了一下返回的数据，正常json字符串后面还带着一个{d:null}。然后打断点调试一下webservice接口，看到返回之前都是正常的，那这个{d:null}是怎么来的呢？

原因分析

上网问了一下度娘，有个大神说得挺清楚的：由于 .net framework3.5 以上添加了对content-type的检查，当ajax发送请求时，如果设置了content-type为application/json，那么请求webservice时，会自动将返回的内容转为json的格式，json的格式iruxia {“d”: “webservice方法返回的字符串内容”}。
